# DeckGenie 产品需求文档

## 1. 产品概述

### 1.1 产品定位

**DeckGenie** 是一个AI驱动的演示文稿自动生成平台，帮助用户将原始内容（文本、PDF、网页链接）快速转化为结构化、视觉化的专业幻灯片。

**核心价值**：
- 节省制作PPT的时间成本（从数小时降至数分钟）
- 提供专业的视觉设计和内容结构建议
- 支持自然语言交互式编辑和优化

**目标用户**：
- 企业员工：需要快速制作工作汇报、项目提案
- 教育工作者：制作课程讲义、教学材料
- 咨询顾问：快速生成客户演示文稿
- 创业者：制作融资路演PPT

### 1.2 产品边界

**系统负责**：
- 解析和理解用户提供的原始内容
- 自动生成演示文稿大纲和页面结构
- 智能选择合适的版式模板
- 生成或推荐配图、图表、图形
- 支持自然语言交互式编辑
- 导出为标准PPTX格式

**系统不负责**：
- 替代用户进行开放式内容研究
- 实现像PowerPoint一样的完整编辑功能
- 支持复杂动画和母版系统
- 提供像素级精确设计工具

### 2.4 AI生成引擎 (AI Generation Pipeline)

**功能描述**：
多Agent协作的智能生成流程：

1. **内容分析Agent**：
   - 理解原始内容的主题、结构、关键信息
   - 识别内容类型（数据、故事、论证、教学等）
   - 提取关键实体和关系

2. **大纲规划Agent**：
   - 生成演示文稿的逻辑结构和页面分配
   - 确定每页的核心信息和辅助信息
   - 规划页面间的逻辑关系

3. **版式匹配Agent**：
   - 根据页面内容类型查询布局DSL库
   - 为每页选择最合适的布局模板
   - 考虑整体视觉连贯性

4. **内容填充Agent**：
   - 将内容分配到布局的各个槽位
   - 优化文字长度和表达方式
   - 确保内容适配布局约束

5. **视觉增强Agent**（核心决策）：
   - 判断每页是否需要视觉元素（图表/图片/图标）
   - **决策流程**：
     - 优先查询资源库中的可复用元素
     - 如果找到匹配元素，复用并修改参数
     - 如果没有合适元素，判断是自绘还是搜图
     - 自绘：适合数据图表、流程图、简单图形
     - 搜图：适合照片、插画、复杂场景
   - 生成或获取视觉元素并集成到页面

6. **质量审核Agent**：
   - 检查内容完整性、视觉一致性
   - 验证布局是否合理、文字是否溢出
   - 评估整体美观度和专业度

**当前状态**：⏳ 规划中
- 需要集成LLM API（OpenAI/DeepSeek/Qwen等）
- 需要设计Agent协作流程和通信协议
- 需要实现ReAct模式的任务执行
- 需要实现视觉增强Agent的三路决策逻辑

### 2.6 模板系统 (Template System)

**核心创新**：通过AI理解截图和描述，自动生成结构化布局模板

**功能描述**：

**A. 布局模板创建（AI驱动）**
- **截图上传**：用户上传参考布局的截图
- **文字描述**：用户提供详细的布局描述（如"左侧图片右侧文字列表形式"）
- **AI解析**：LLM分析截图的视觉布局 + 结合文字描述
- **生成布局DSL**：输出JSON格式的"布局描述语言"，包含：
  - 页面区域划分（左右分栏、上下布局等）
  - 内容槽位定义（标题、正文、图片、列表等）
  - 尺寸和位置约束
  - 样式参数（字体、颜色、间距等）
- **存储**：布局DSL存入数据库，可被Agent调用

**示例布局DSL**：
```json
{
  "layoutId": "left-image-right-list",
  "name": "左图右列表",
  "description": "左侧大图，右侧标题+要点列表",
  "regions": [
    {
      "id": "left-image",
      "type": "image",
      "bounds": {"x": 0, "y": 0, "width": 0.45, "height": 1}
    },
    {
      "id": "right-content",
      "type": "container",
      "bounds": {"x": 0.5, "y": 0, "width": 0.5, "height": 1},
      "children": [
        {"type": "title", "maxLines": 2},
        {"type": "bullet-list", "maxItems": 5}
      ]
    }
  ]
}
```

**B. 布局匹配和应用**
- **智能匹配**：Agent根据页面内容类型自动选择最合适的布局
- **布局库管理**：按场景分类（封面页、内容页、对比页、总结页等）
- **布局预览**：用户可浏览和手动选择布局

**当前状态**：🚧 UI已完成，核心功能待实现

### 2.7 资源库 (Asset Library)

**核心创新**：从现有PPT中智能提取可复用元素

**功能描述**：

**A. PPT元素提取（AI驱动）**
- **PPT上传**：用户上传现有的PPT文件（如公司500页样式库）
- **智能解析**：系统自动识别和提取可复用元素：
  - 图表（柱状图、饼图、折线图、雷达图等）
  - 表格（数据表格、对比表格）
  - 插画和图标
  - 信息图（流程图、组织架构图、时间轴等）
  - 装饰元素（边框、分隔线、背景图案）
- **SVG转换**：将提取的元素转换为SVG格式
- **元数据标注**：
  - 元素类型（图表/表格/插画/图标）
  - 风格标签（商务/科技/简约/卡通）
  - 颜色主题
  - 适用场景
- **存储**：SVG文件存储在本地文件系统，元数据存入数据库

**B. 元素复用策略（Agent决策）**
在生成PPT时，视觉增强Agent会根据内容需求做出决策：

1. **复用已有元素**：
   - 从资源库中查找匹配的元素
   - 修改颜色、尺寸、数据以适配当前内容
   - 优点：风格统一、符合企业规范

2. **自绘新元素**：
   - 使用代码生成简单图表（基于D3.js/Chart.js）
   - 绘制基础图形和流程图
   - 优点：完全定制、精确匹配需求

3. **网络搜图**：
   - 调用图片搜索API（Unsplash/Pexels等）
   - 使用AI图像生成（DALL-E/Stable Diffusion）
   - 优点：内容丰富、视觉冲击力强

**决策逻辑示例**：
```
如果需要展示"市场份额对比"：
  → 优先查找资源库中的饼图模板
  → 如果找到匹配风格的，复用并修改数据
  → 如果没有，自绘简单饼图
  → 如果需要特殊效果，考虑AI生成

如果需要"团队照片"：
  → 优先使用用户上传的图片
  → 如果没有，从网络搜索相关图片
  → 不使用自绘（照片类内容不适合）
```

**C. 资源管理**
- **分类浏览**：按类型、风格、颜色筛选
- **搜索功能**：关键词搜索、相似度搜索
- **收藏和标签**：用户可收藏常用元素、添加自定义标签
- **批量导入**：支持一次性上传多个PPT文件进行批量提取

- **LLM成本**：大量API调用可能导致成本过高

- **用户学习成本**：用户可能不理解如何有效使用AI功能

- **内容版权**：用户上传的内容或生成的图片可能涉及版权问题

## 8. 核心技术创新详解

### 8.1 AI驱动的布局模板生成

**问题背景**：
传统PPT工具需要设计师手工制作模板，成本高、周期长。DeckGenie通过AI理解截图和描述，自动生成结构化布局模板。

**技术流程**：

```
用户输入
  ├─ 截图上传（参考布局的视觉示例）
  └─ 文字描述（"左侧图片右侧文字列表形式"）
      ↓
LLM多模态分析
  ├─ 视觉理解：识别截图中的区域划分、元素位置
  ├─ 文字理解：解析用户的布局意图和约束
  └─ 结合分析：生成结构化的布局描述
      ↓
布局DSL生成
  ├─ 区域定义（regions）：位置、尺寸、类型
  ├─ 槽位定义（slots）：内容类型、约束条件
  ├─ 样式参数：字体、颜色、间距
  └─ 输出JSON格式
      ↓
存储和应用
  ├─ 存入数据库（可被Agent查询）
  └─ 渲染引擎根据DSL生成实际页面
```

**布局DSL示例**：
```json
{
  "layoutId": "comparison-two-column",
  "name": "双栏对比",
  "category": "content",
  "description": "左右两栏，适合对比展示",
  "canvas": {"width": 1920, "height": 1080},
  "regions": [
    {
      "id": "left-column",
      "type": "container",
      "bounds": {"x": 60, "y": 150, "width": 850, "height": 800},
      "style": {"background": "#f8f9fa", "border": "2px solid #e9ecef"},
      "slots": [
        {"id": "left-title", "type": "text", "maxLines": 2, "fontSize": 32},
        {"id": "left-content", "type": "bullet-list", "maxItems": 5}
      ]
    },
    {
      "id": "right-column",
      "type": "container",
      "bounds": {"x": 1010, "y": 150, "width": 850, "height": 800},
      "style": {"background": "#f8f9fa", "border": "2px solid #e9ecef"},
      "slots": [
        {"id": "right-title", "type": "text", "maxLines": 2, "fontSize": 32},
        {"id": "right-content", "type": "bullet-list", "maxItems": 5}
      ]
    }
  ]
}
```

**关键技术点**：
- **多模态LLM**：需要支持图像理解的模型（GPT-4V、Claude 3、Qwen-VL等）
- **Prompt工程**：设计精确的提示词，引导LLM输出标准化的JSON格式
- **DSL验证**：确保生成的布局DSL符合规范，可被渲染引擎解析
- **布局库管理**：分类、搜索、版本控制

### 8.2 PPT元素智能提取和复用

**问题背景**：
企业通常有大量现成的PPT资源（如公司500页样式库），但这些资源难以被系统化复用。DeckGenie通过AI提取和标注，将这些资源转化为可编程的元素库。

**技术流程**：

```
PPT文件上传
  ↓
PPTX解析
  ├─ 解压PPTX文件（本质是ZIP）
  ├─ 解析XML结构（slides/slide1.xml等）
  └─ 提取形状、图片、图表数据
      ↓
元素识别和分类
  ├─ 图表识别：检测chart.xml，提取图表类型和数据
  ├─ 表格识别：检测table元素，提取行列结构
  ├─ 图片识别：提取图片文件，使用AI识别内容类型
  ├─ 形状识别：提取矢量形状，判断是图标还是装饰
  └─ 文本框过滤：排除纯文本内容
      ↓
SVG转换
  ├─ 图表：转换为SVG格式，保留数据结构
  ├─ 形状：转换为SVG路径
  ├─ 图片：保留原格式或转PNG
```

```python
    """
    视觉增强Agent的决策逻辑
    """

    if matching_assets:
        # 找到匹配元素，复用并修改
        best_match = matching_assets[0]
        modified_asset = modify_asset_parameters(
            asset=best_match,
            new_data=context.data,
            new_colors=context.colors
        )
        return {"method": "reuse", "asset": modified_asset}

    # 2. 判断是否适合自绘
    if content_type in ["bar_chart", "line_chart", "pie_chart", "flowchart"]:
        # 简单图表，自绘
        return {"method": "generate", "tool": "chart.js"}

    # 3. 网络搜图或AI生成
    if content_type in ["photo", "illustration", "background"]:
        return {"method": "search", "source": "unsplash"}

    # 4. 默认：不添加视觉元素
    return {"method": "none"}
```

### 8.3 技术挑战和解决方案

**挑战1：布局DSL的标准化**
- 问题：不同类型的布局差异很大，如何设计统一的DSL？
- 方案：采用分层设计，基础层定义通用属性（位置、尺寸），扩展层定义特定类型的属性

**挑战2：PPT格式的复杂性**
- 问题：PPTX格式复杂，包含大量Office特有的属性和效果
- 方案：只提取核心视觉元素，忽略复杂动画和特效；提供降级方案

**挑战3：元素复用的准确性**
- 问题：如何确保复用的元素真正匹配当前内容？
- 方案：多维度匹配（类型+风格+颜色+场景），引入相似度评分，低于阈值则不复用

**挑战4：SVG兼容性**
- 问题：SVG在不同浏览器和导出工具中表现可能不一致
- 方案：使用标准SVG特性，避免浏览器特定扩展；充分测试导出流程
